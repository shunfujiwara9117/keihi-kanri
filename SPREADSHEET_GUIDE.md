# 📊 Googleスプレッドシート機能 完全ガイド

確定申告用の経費管理をGoogleスプレッドシートで行うための完全ガイドです。

## 🎯 概要

2つの方法でGoogleスプレッドシートを作成できます：

| | 方法1: 自動作成 | 方法2: テンプレートコピー |
|---|---|---|
| **データ入力** | 自動（登録済みデータ） | 手動 |
| **セットアップ時間** | 10分（初回のみ） | 5分（初回のみ） |
| **使いやすさ** | ボタン1つ | ボタン1つ |
| **おすすめ** | データ量が多い人 | シンプルに使いたい人 |

---

## 📋 方法1: 自動作成（データ自動入力）

### 特徴

- ✅ ボタン1つでスプレッドシート作成
- ✅ 登録済みの経費データが**自動的に入力**される
- ✅ 新しいタブで自動的に開く
- ✅ 完全自動化

### セットアップ手順（初回のみ・10分）

詳細は [SETUP_QUICK_GUIDE.md](./SETUP_QUICK_GUIDE.md) を参照してください。

#### ステップ1: Google Apps Scriptプロジェクトを作成

1. https://script.google.com/ を開く
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「経費管理API」に変更

#### ステップ2: コードをコピー＆貼り付け

1. `API-script-v2.gs` ファイルを開く
2. 全部選択（Cmd+A）してコピー（Cmd+C）
3. Google Apps Scriptのエディタに貼り付け
4. 保存（Cmd+S）

#### ステップ3: Webアプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」
2. 「種類の選択」⚙️ → 「ウェブアプリ」
3. 設定：
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. 「デプロイ」をクリック
5. 権限の承認（「詳細」→「移動」→「許可」）
6. **ウェブアプリのURL**をコピー

#### ステップ4: URLを設定

`.env.local` ファイルに以下を追加：

```bash
NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL=ここにコピーしたURL
```

#### ステップ5: サーバーを再起動

```bash
npm run dev
```

### 使い方

1. アプリで経費データを登録
2. 「📝 自動作成」ボタンをクリック
3. 新しいタブでGoogleスプレッドシートが開く
4. **データが既に入力されている！**

---

## 📋 方法2: テンプレートコピー（手入力）

### 特徴

- ✅ セットアップが簡単（5分）
- ✅ 空のテンプレートをコピー
- ✅ 手動で経費を入力
- ✅ シンプルで確実

### セットアップ手順（初回のみ・5分）

詳細は [CREATE_TEMPLATE.md](./CREATE_TEMPLATE.md) を参照してください。

#### ステップ1: テンプレートを作成

1. https://script.google.com/ を開く
2. 「新しいプロジェクト」をクリック
3. `template-script.gs` のコードをコピー＆貼り付け
4. 「createTemplate」関数を実行
5. **テンプレートID**をコピー

#### ステップ2: IDを設定

`.env.local` ファイルに以下を追加：

```bash
NEXT_PUBLIC_GOOGLE_SHEETS_TEMPLATE_ID=ここにコピーしたID
```

#### ステップ3: テンプレートを共有設定

1. 作成されたスプレッドシートを開く
2. 「共有」→「リンクを知っている全員が閲覧可能」に変更

### 使い方

1. 「📋 テンプレートをコピー」ボタンをクリック
2. Googleドライブにコピーが作成される
3. 手動で経費を入力

## 🔧 トラブルシューティング

### Q: ボタンをクリックしても何も起こらない

**原因**: 環境変数が設定されていない

**解決策**:
1. `.env.local` ファイルを確認
2. URLまたはIDが正しく設定されているか確認
3. サーバーを再起動（`npm run dev`）

### Q: エラーが出る

**原因1**: Google Apps Scriptのデプロイが正しく完了していない

**解決策**:
1. Google Apps Scriptを開く
2. 「デプロイ」→「デプロイを管理」
3. 「新バージョン」を選択して再デプロイ

**原因2**: 権限の承認が完了していない

**解決策**:
1. Google Apps Scriptで関数を実行
2. 「詳細」→「移動」→「許可」をクリック

### Q: 新しいタブが開かない

**原因**: ポップアップブロック

**解決策**:
1. ブラウザのアドレスバー右側のアイコンを確認
2. ポップアップを「常に許可」に設定
3. もう一度ボタンをクリック
