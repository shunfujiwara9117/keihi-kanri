# 🚀 クイックセットアップガイド

## 方法1: 自動化セットアップ（10分）

### ステップ1: Google Apps Script を開く

1. ブラウザで https://script.google.com/ を開く
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「経費管理API」に変更（左上の「無題のプロジェクト」をクリック）

### ステップ2: コードをコピー

1. `GOOGLE_APPS_SCRIPT.md` ファイルを開く
2. ```javascript``` で始まるコード全体をコピー（約300行）
3. Google Apps Scriptのエディタに貼り付け（既存のコードを全て置き換え）
4. 「保存」アイコンをクリック（💾）

### ステップ3: デプロイ

1. 画面右上の「デプロイ」→「新しいデプロイ」をクリック
2. 「種類の選択」（歯車アイコン⚙️）をクリック
3. 「ウェブアプリ」を選択
4. 以下を設定：
   ```
   説明: 経費管理API v1
   次のユーザーとして実行: 自分（あなたのメールアドレス）
   アクセスできるユーザー: 全員
   ```
5. 「デプロイ」ボタンをクリック

### ステップ4: 権限の承認

初回のみ、権限の承認が必要です：

1. 「アクセスを承認」をクリック
2. Googleアカウントを選択
3. 警告画面が出たら：
   - 「詳細」をクリック
   - 「経費管理API（安全ではないページ）に移動」をクリック
   - 「許可」をクリック

### ステップ5: ウェブアプリURLをコピー

デプロイが完了すると、**ウェブアプリのURL**が表示されます：

```
https://script.google.com/macros/s/AKfycbz.../exec
```

このURL全体をコピーしてください（**必ず `/exec` まで含める**）

### ステップ6: URLを設定

#### 方法A: コマンドで設定（簡単）

ターミナルで以下を実行（URLを自分のものに置き換えてください）：

```bash
echo 'NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec' > /Users/fujiwarashun/0204youtube/keihi-kanri/.env.local
```

#### 方法B: 手動で設定

1. ファイルを開く：
   ```
   /Users/fujiwarashun/0204youtube/keihi-kanri/.env.local
   ```

2. 以下を追加（URLを自分のものに置き換え）：
   ```
   NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec
   ```

3. 保存

### ステップ7: サーバーを再起動

ターミナルで：

```bash
# Ctrl+C でサーバーを停止
# その後、再起動
npm run dev
```

### ステップ8: テスト

1. ブラウザで http://localhost:3000 を開く
2. 「📝 スプレッドシート作成」ボタンをクリック
3. 新しいタブでGoogleスプレッドシートが開けばOK！

## トラブルシューティング

### Q: ボタンをクリックしても何も起こらない

**原因1: .env.localが読み込まれていない**

確認：
```bash
cat /Users/fujiwarashun/0204youtube/keihi-kanri/.env.local
```

URLが表示されない場合 → 方法Bで手動設定

**原因2: サーバーが再起動されていない**

解決：
```bash
# Ctrl+C でサーバーを停止
npm run dev
```

### Q: 「アクセスが拒否されました」と表示される

**原因: 権限の承認が完了していない**

解決：
1. Google Apps Scriptに戻る
2. デプロイを削除
3. もう一度「新しいデプロイ」を実行
4. 今度は確実に「許可」をクリック

### Q: 新しいタブが開かない

**原因: ポップアップブロック**

解決：
1. ブラウザのアドレスバー右側のアイコンを確認
2. ポップアップを「常に許可」に設定
3. もう一度ボタンをクリック
